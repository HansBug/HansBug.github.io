## 为什么要先想“长期写下去”

很多个人博客失败，并不是因为没有技术能力，而是因为一开始就把注意力放错了。界面做得很满、功能堆得很杂、部署流程很脆弱，最后每次想发一篇文章都像在做小型发布会。

如果目标是长期写作，那么博客首先应该是一个**低摩擦的工作台**。写一篇、改一篇、补一条路线、加一个标签，都应该是顺手动作，而不是额外项目。

## 我对博客底座的最低要求

第一，结构要清楚。至少要把文章、标签、归档、路线和联系入口分开，否则后续内容一多就会混成时间流。

第二，部署要稳定。个人博客最怕“过几个月完全忘了怎么发”。如果能做到 push 到主分支就自动发布，维护成本会小很多。

第三，内容要能互相引用。单篇文章解决的是一个局部问题，但真正有价值的是：以后能不能从这篇文章继续跳到同主题的其他内容。

## 为什么静态站点依然是很好的选择

对个人博客来说，静态站点的核心价值不在于“更潮”，而在于边界清晰。没有数据库、没有运行时状态、没有复杂后端，故障面非常可控。

尤其是在 GitHub Pages 这种托管场景里，静态站点意味着：

- 发布链路更短
- 回滚更简单
- 仓库本身就是内容源
- 本地开发和线上结果更接近

## 第一版到底该做什么

我更倾向于把第一版控制在这几件事：

1. 文章系统可写可发。
2. 标签和归档可以用。
3. 至少有一层阅读路线。
4. 联系方式明确可见。

等这些稳定下来，再去做评论、搜索增强、国内镜像或者更复杂的视觉效果。这样整个博客会更像一个长期资产，而不是一次性作品。
